# 점검서류 사진 분류기

점검 서류 사진을 업로드하면 **Claude Vision**이 각 사진에서 **사업명**과 **서류 제목**을 읽어,
자동으로 **사업명별 폴더**에 분류하고 **서류 제목을 파일명**으로 만들어 저장합니다.
핸드폰 홈 화면에 앱으로 설치해서 쓸 수 있습니다(PWA).

## 핵심 규칙

- 사진 안에 **사업명이 보이면** → 그 사업명으로 새 그룹(폴더)을 만듭니다. (보통 **표지**)
- 사진 안에 **사업명이 없으면** → **바로 앞 표지의 사업명**에 이어지는 사진으로 보고 같은 폴더에 넣습니다.
- 그래서 **업로드 순서가 중요합니다.** 표지 → 그 안의 내용 사진들 순서로 올리세요.

## 사용법

1. 앱을 엽니다. (데스크톱은 `index.html` 더블클릭, 핸드폰은 아래 "홈 화면 설치" 참고)
2. **Anthropic API 키**를 입력합니다. (브라우저에만 저장되며 서버로 전송되지 않습니다.)
   - 키 발급: https://console.anthropic.com/
3. 서류 사진을 **드래그하거나 선택**해서 업로드합니다. (여러 장 가능, 순서 유지)
4. **[분석 시작]** — 각 사진을 Claude가 분석합니다.
5. 결과 표에서 **사업명·제목·표지 여부를 확인/수정**합니다. (표지 행은 노란색)
6. **[사업명 폴더로 저장]** — 사업명별로 정리해 저장합니다.

## 저장 결과 예시

```
├─ A아파트 소방점검/
│  ├─ 표지.jpg
│  ├─ 소방시설 점검표.jpg
│  └─ 전기안전 점검 결과서.jpg
└─ B빌딩 정기점검/
   ├─ 표지.jpg
   └─ 승강기 점검표.jpg
```

## 📱 핸드폰 홈 화면에 앱으로 설치하기 (PWA)

### 1) 먼저 인터넷 주소로 배포 (최초 1회)
GitHub Pages로 무료 배포됩니다.
1. 이 PR을 `main`에 병합합니다.
2. 저장소 **Settings → Pages → Build and deployment → Source** 를 **"GitHub Actions"** 로 설정합니다.
3. 배포가 끝나면 주소가 생깁니다: `https://zianocom.github.io/photo-sorter/`
   - (병합 전에 먼저 써 보려면 Settings → Pages 에서 "Deploy from a branch" 로 이 브랜치를 선택해도 됩니다.)

### 2) 폰에서 홈 화면에 추가
위 주소를 폰 브라우저로 연 뒤:
- **아이폰(Safari)**: 공유 버튼 → **"홈 화면에 추가"**
- **안드로이드(Chrome)**: 우측 상단 ⋮ → **"홈 화면에 추가"**

그러면 바탕화면에 📄 아이콘이 생기고, 탭하면 주소창 없이 앱처럼 열립니다.

## 저장 방식

- **데스크톱 Chrome/Edge**: 폴더를 직접 선택해 `사업명/제목.jpg`로 저장합니다.
- **모바일·기타 브라우저**: 사업명 폴더 구조를 그대로 담은 **ZIP 파일 하나**로 내려받습니다. 압축을 풀면 사업명별로 정리됩니다.

## 참고

- 이미지는 전송 전 자동으로 축소(최대 1568px)되어 분석 속도와 비용을 줄입니다. **저장되는 파일은 원본**입니다.
- 같은 폴더에 동일한 제목이 있으면 `제목_2.jpg`처럼 번호가 붙습니다.
- API 키는 브라우저 localStorage에만 저장되며 서버로 전송되지 않습니다.
- 모델은 기본값(`claude-sonnet-5`)을 권장하며, 설정에서 변경할 수 있습니다.
